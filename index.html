Introduction to TypeScript

TypeScript serves as an extension to JavaScript by providing type checking and a way to define types. By doing this TypeScript gives the developer the option to add more steps, to do the same thing in their software. The best part about TypeScript though, it still follows JavaScript's nonsensical semantics for consistency.

Defining Types

Below is your typical creation of an object in JavaScript.

    
        const student = {
            name: "Aurora",
            id: 0,
        };

With TypeScript, you can do an interface declaration. This will provide a defined type.

        
            interface Student {
                name: string;
                id: number;
            }

After declaring an interface you can then create an object that is defined by the interface. This will provide type checking.

        
            const Student {
                name: "Aurora",
                id: 0,
            };

Composing Types

TypeScript allows the manifestation of complex types by combining simple ones. This is through unions and generics.

Unions

Unions offer the ability to declare a type that can be one of many things. A simple example as shown below shows that a boolean can either be true or false.

        
            type absent = true | false;

This also works for other types, below is an example for class rankings.

        
            type classRanking = "valedictorian" | "salutatorian" | "Not up to snuff" | "deplorable";
