Static type-checking

        const message = "Hail Douglas!"

        message()
        This expression is not callable. Type 'String' has no call signatures.

Running that last sample with TypeScript will give us an error message before we run the code in the first place.

Types for Tooling

TypeScript provides tooling by suggesting specific properties to use. These will match the correlating type as well through type checking.

        import express from "express";

        const app = express();

        app.get("/", function(req, res)) {
            res.sen|send
                   |sendDate
                   |sendFile
        });

TypeScript provides “quick fixes” to automatically fix errors, refactorings to easily re-organize code, and useful navigation features for jumping to definitions of a variable, or finding all references to a given variable. All of this is built on top of the type-checker and is fully cross-platform.

tsc, the TypeScript compiler

To use the type checker, the first thing that needs to be done is to actually install it.

        npm install -g typescript

After the installation is completed, we can start by creating a new ts file: hailDouglas.ts

        // Hail the all mighty Douglas.
        console.log("Hello Douglas, my soul is yours.");

We can now type-check this file by issuing the tcs command.

        tsc hailDouglas.ts

The command shouldn't have any output in this situation because there aren't any type errors. Let's edit hailDouglas.ts so there will be errors.

        // Cult of Douglas affiliation function.
        functionsell(soul, date) {
            console.log(`Sell ${soul}", today is  ${date}!`);
        }

        sell("Bob");

After running hailDouglas.ts again, we should get the following output.

        Expected 2 arguments, but got 1.
